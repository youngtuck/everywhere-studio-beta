# EVERYWHERE(TM) Glossary

**Version:** 6.0
**Last Updated:** February 17, 2026

---

## A-B

**Betterish**
Objective quality scoring system with a 450+ threshold required for publication. Also the SBU agent who speaks last in evaluation sequences. "Does this make you want to use it?"

**Blocking Authority**
Charlie's power to prevent delivery. Sara cannot ship without Charlie's verification. Charlie cannot be overridden. If Charlie blocks, nothing ships.

**BOH (Back of House)** (NEW v5.4)
Development workstream. Where Riley, the build team, and Friday dev sessions operate. BOH figures out what works, what doesn't, what's good, what's bad. Then pushes to FOH.

**Build Master**
Agent role (Riley) responsible for tracking variants, managing Friday builds, and maintaining version integrity across all EVERYWHERE editions.

---

## C

**Cage Match**
Structured intellectual combat between competing ideas. Four rounds: Steel Man, Direct Confrontation, Stress Testing, Synthesis Check. Facilitated by Dana White.

**Client Fact Sheet**
Pre-production artifact containing verified client details. Reed cannot begin draft without a confirmed fact sheet.

**Composer**
Mark's role in the orchestra metaphor. Creates the ideas, vision, and creative direction.

**Composer Override**
When Mark provides text, it replaces the draft immediately. No blending. No asking if he's sure. Core behavior across all variants.

**Conductor**
Sara's role in the orchestra metaphor. Orchestrates the team, coordinates execution, ensures delivery, tracks learning state.

**Core**
The foundational layer of capabilities present in all EVERYWHERE variants: Interaction Modes, One-Question-At-A-Time, Voice DNA, Quality Checkpoints, Betterish Scoring, Navigation Triad, Learning Mode, UX Review.

---

## D-E

**DISCOVER**
Navigation system -- the table of contents. Browse all capabilities, find what exists, explore deep dives on any feature. Part of the Navigation Triad.

**Don't Ask Rule**
When the path is clear, execute. Would a trusted Chief of Staff ask this question, or just handle it?

**Dual-Output**
Reed's automatic production of two versions for every Sunday Story: Article (polished prose) and Podcast Script (conversational). No separate request needed.

**Euler Test**
Category validation framework: Does this solve the problem with mathematical elegance?

---

## F

**First Listener**
Reed's role. The first to hear Mark's ideas and capture them on the score.

**Five Layers**
Josh Lowman's Category Leadership framework: Central Idea, Mic Drop POV, Irreplaceable Product, Confident Brand, Visible Founder.

**FOH (Front of House)** (NEW v5.4)
Product workstream. The app, onboarding, customer experience. FOH users get buttons. BOH users get triggers. If it's not simple enough for FOH, it's a fail.

---

## I

**Interaction Modes**
Five modes governing how conversations flow (Core capability): Brainstorm, Decision, Information, Task, Open.

**Interest Graph**
Distribution based on behavior and content quality, not follower count.

**Interest Graph Framework (IGF)**
Four filters: Category Signal (Josh), Worldview Marker (Lea), Conversion Architecture (Guy), Qualification Filter (Ward).

**Inverse Principle** (NEW v5.4)
The more complex the system gets, the harder we work to make it simple. Complexity is BOH's problem. Simplicity is FOH's requirement. If it's not simple, it's a fail.

---

## L-M

**Learning Mode** (NEW v5.4)
System Mode 7. SODOTU-based training. Sande demonstrates, guides practice, validates mastery. You learn EVERYWHERE by using EVERYWHERE, not by reading about it. See also: SODOTU, Sande.

**Leverage Point Analysis**
Fish's identification of where small effort creates outsized results.

**BOH (formerly Studio)**
The full EVERYWHERE variant with all 40 agents, named personalities, team routing, SBU, Interest Graph Framework, and Special Forces. Where new features get tested.

**Methodology Pack**
Client-specific framework added to Solo edition to create specialized versions.

**Mic Drop POV**
Strategic narrative formula: [Customer] has a [Great Ambition] but [Enemy] stands in the way. We [Vanquish Enemy] by [How].

**Musicians**
The agents in the orchestra metaphor. Each plays their instrument.

---

## N-O

**Navigation Triad** (NEW v5.4)
Three Core systems that eliminate memorization: DISCOVER (table of contents), WHAT'S NEXT (suggestion engine), LEARN (training ground). Nobody memorizes triggers.

**OMV**
Objectives, Metrics, Value -- Victor's precision tool.

**One-Question-At-A-Time**
Core UX principle: Never ask more than one question per turn.

**OPAT**
Outcome, Purpose, Action, Timing -- Victor's strategic framing framework.

**Composed Intelligence**
The core concept of EVERYWHERE. Multiple specialized AI agents working together. The difference between a one-person band and a symphony.

**Output System**
Complete package for producing a specific content type. Nine total.

---

## Q

**Quality Checkpoints**
Seven mandatory checkpoints: Checkpoint 0 (Echo/deduplication), Checkpoint 1 (Priya/research), Checkpoint 2 (Jordan/voice), Checkpoint 3 (David/engagement), Checkpoint 4 (Elena/SLOP), Checkpoint 5 (Natasha/editorial + Stranger Test), Checkpoint 6 (Marcus + Marshall/perspective + NVC). Checkpoints are blockers, not suggestions.

**Quick Review**
System Mode 5. Lightweight SBU check: Jordan + David + Natasha + relevant SBU member. Trigger: "SBU?"

**Quick UX** (NEW v5.4)
Abbreviated UX Review running checks 1 and 2 only (Strategy Alignment + User Journey). Trigger: "Quick UX"

---

## S

**Sande** (NEW v5.4)
The Trainer. Agent #40. Owns Learning Mode execution. Reports to Natasha. Works with Diane (Sande teaches, Diane documents). Runs SODOTU sessions.

**SLOP**
What Dr. Elena Vasquez detects: Superfluity, Loops, Overwrought prose, Pretension.

**SBU (Strategic Business Unit)**
Nine agents ensuring content works for strangers AND handles business outcomes. Members: Victor, Josh, Lea, Guy, Ward, Fish, Basil Drake, Dana White, Betterish.

**SODOTU** (NEW v5.4)
See One, Do One, Teach One. Medical training methodology adapted for composed intelligence. The three phases of Learning Mode. See also: Learning Mode, Sande.

**Solo (formerly Vanilla)**
Core-only variant with no agent names visible. Functions presented instead of personalities.

**Stranger Test**
Checkpoint 5 check: If someone who has never heard of Mark, EVERYWHERE, or Coastal Intelligence reads this cold -- do they understand what each thing IS within 30 seconds of first mention?

**Structured Intelligence**
Synth's output format for processing call transcripts.

**System Modes**
Seven modes governing how EVERYWHERE operates: (1) Content Production, (2) Path Determination, (3) Decision Validation, (4) The Stress Test, (5) Quick Review, (6) UX Review, (7) Learning Mode.

---

## T-V

**The Stress Test**
System Mode 4. Six-phase naming/positioning validation: Requirements, Generation, Research, Cage Match, Selection, Build-Out.

**UX Review** (NEW v5.4)
System Mode 6. Five sequential checks for digital assets: Strategy Alignment, User Journey, Engagement & Conversion, Copy & Voice, Technical & Visual. Led by Christopher Kowalski. Trigger: "Run this through UX"

**Variant**
A specific edition of EVERYWHERE built on Core: BOH (full system, 40 agents), Solo (core only, no names), Coaches (Solo + methodology packs), FOH/App (customer-facing).

**Voice DNA**
Captured patterns of how someone communicates. Three layers: Voice Markers (patterns), Value Markers (beliefs), Personality Markers (assessment-driven).

---

## W

**WHAT'S NEXT**
Navigation system -- the suggestion engine. Follows you contextually, surfaces the right trigger at the right moment. Sara watches context and offers relevant next steps. Part of the Navigation Triad.

**Workflow**
The sequence of steps within an Output System.

---

## Usage Notes

**When introducing EVERYWHERE to someone new:**
Start with Composed Intelligence, then explain Core, then describe the specific Variant they'll use.

**When explaining why this is different:**
Lead with Interest Graph (the world changed), then Voice DNA (we solve the authenticity problem), then Quality Checkpoints (we enforce standards).

**When talking about business value:**
SBU, OPAT, Leverage Point Analysis, Conversion Architecture.

**When teaching someone the system:**
Use the Navigation Triad: DISCOVER to browse, What's Next to suggest, Sande to train.
