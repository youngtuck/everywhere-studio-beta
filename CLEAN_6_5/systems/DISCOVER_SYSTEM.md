# DISCOVER -- Capability Discovery System

**Version:** 6.0
**Last Updated:** February 17, 2026
**Status:** Core Capability

---

## WHAT IS DISCOVER

DISCOVER surfaces what EVERYWHERE can do. Users don't need to memorize commands, read manuals, or know system internals.

**The insight:** Users get stuck because they didn't know a capability existed, not because they couldn't use it.

Type discover anytime. The system figures out the rest.

---

## DISCOVER VS. WHAT'S NEXT VS. LEARNING MODE (v5.4)

| System | Purpose | When to Use |
|--------|---------|-------------|
| **DISCOVER** | Show all capabilities (the map) | "What can this do?" |
| **WHAT'S NEXT** | Show relevant next step (the compass) | "What should I do now?" |
| **LEARNING MODE** | Learn by doing (the training ground) | "How does this actually work?" |

DISCOVER is for exploration. What's Next is for direction. Learning Mode is for mastery.

They complement each other:
- Lost? -> DISCOVER
- Stuck? -> WHAT'S NEXT
- New to something? -> LEARNING MODE

## SMART ROUTING

When user says discover how to [anything], the system:

1. Identifies which capability area applies
2. Explains the relevant process
3. Offers to begin OR offers Learning Mode

**Examples that work:**

- Discover how to write a blog post -> Essay writing process
- Discover how to make a video -> Video script creation
- Discover how to test my decision -> Decision Validation
- Discover how to name something -> The Stress Test
- Discover how to improve this -> Quality and improvement options
- Discover how to check my blind spots -> Perspective analysis
- Discover how quality works -> Quality checkpoints explanation
- Discover how to communicate a decision -> Communication cascade
- Discover how to analyze a market -> Category and positioning tools
- Discover how to research something -> Research capabilities
- Discover how to quick check something -> Quick Review
- Discover what's next -> Explains the What's Next system
- Discover how to learn something -> Explains Learning Mode (NEW v5.4)
- Discover how to review UX -> Explains UX Review workflow (NEW v5.4)

The system always figures it out. If unclear, it asks one clarifying question.

**NEW v5.4:** After explaining any capability, DISCOVER now offers: "Want to see it in action? Say 'teach me [X]'"

## DEEP DIVE: LEARNING MODE (NEW v5.4)

```
DISCOVER: Learning Mode (SODOTU)

Learn EVERYWHERE by using EVERYWHERE.

THE METHODOLOGY
See One, Do One, Teach One -- adapted from medical training.
You don't learn surgery by reading about it.
You don't learn composed intelligence by reading about it either.

THREE PHASES

Phase 1: SEE ONE (Watch)
-> System runs the full capability on a real example
-> Narrates what's happening and why at each step
-> You watch. No decisions required.
-> "Starting a Sunday Story. Watch how this works..."

Phase 2: DO ONE (Practice)
-> You bring your own topic or task
-> System guides you through each step
-> Training wheels on -- guardrails catch mistakes
-> "Your turn. What decision do you want to validate?"

Phase 3: TEACH ONE (Master)
-> System asks you to explain it to someone new
-> Validates your understanding, fills gaps
-> This is where it sticks.
-> "Explain to a new user how quality checkpoints work."

TRIGGERS
-> "Teach me [X]" -- Full SODOTU sequence
-> "Show me [X]" -- SEE ONE only
-> "Walk me through [X]" -- SEE ONE only
-> "Let me try [X]" -- DO ONE only
-> "I want to learn [X]" -- Full SODOTU sequence

WORKS WITH EVERYTHING
-> System modes, workflows, quality checkpoints, frameworks
-> "Teach me Decision Validation"
-> "Show me how a Sunday Story gets produced"
-> "Teach me Red Teaming"
-> "Show me how Betterish scoring works"

You control the pace. Stop any time. No grades.
```

## DEEP DIVE: WHAT'S NEXT

```
DISCOVER: What's Next

Workflow guidance that knows where you are.

THE DIFFERENCE
DISCOVER = map (here's everywhere you could go)
What's Next = compass (here's where you should probably go)
Learning Mode = training ground (here's how it actually works)

TWO MODES

Reactive (you ask):
-> "What's next"
-> "Now what"
-> "Where was I"
-> "What should I do"
-> "I'm stuck"

Proactive (system notices):
-> You're circling the same topic
-> A phase completed but you haven't moved forward
-> You skipped a step that'll cause problems
-> You seem frustrated
-> The scope expanded without acknowledgment
-> You're in a loop

WHAT IT Task MasterKS
-> Topics you've touched
-> Current workflow phase
-> Deliverables in progress
-> Checkpoints completed
-> Time since progression

WHAT IT DOESN'T DO
-> Replace your judgment
-> Force workflows
-> Nag (rate-limited, dismissable)
-> Override Composer decisions

HOW TO USE
Just work normally. Say "what's next" when you want direction.
The system will also notice if you're stuck and offer help.

To dismiss proactive suggestions:
-> "I know" -- backs off for the session
-> "Not now" -- backs off temporarily
-> Just ignore it -- backs off
```

## DEEP DIVE: COMMUNICATION CASCADE

```
DISCOVER: Communication Cascade

One decision -> Messages for every audience.

WHEN IT ACTIVATES
After Decision Validation returns GREEN and you commit.

WHAT HAPPENS
The validated decision becomes the seed. The system produces:

1. Board version -- Executive summary, strategic rationale
2. Investor version -- Financial implications, growth story
3. Team version -- What changes, what's expected
4. Market version -- External positioning, customer message
5. Press version -- Quotable, newsworthy framing

Each version passes through quality checkpoints before delivery.

WHY IT MATTERS
Most leaders make a decision, then scramble to explain it.
This builds communication into the process.
One idea. Examined. Communicated everywhere.
```

## DEEP DIVE: QUALITY SCORING

```
DISCOVER: Quality Scoring (Betterish)

Removes subjectivity from "is this good enough?"

THE SCALE (0-600 points)

0-299 (0-49%) -- Needs work
Major issues. Rethink approach.

300-449 (50-74%) -- Getting there
Promising but not ready. Targeted improvements needed.

450-549 (75-91%) -- Ready to publish
Solid work. Ship or polish further.

550-600 (92-100%) -- Exceptional
Outstanding. Publish immediately.

HOW IT WORKS

Base Score (360 points max):
- Unique -- Is this fresh?
- Compelling -- Does it grab attention?
- Sustainable -- Will it last?
- Believable -- Is it credible?

Content Bonus (240 points max):
- Varies by content type
- Essays: storytelling depth, insight density
- Podcasts: audio flow, hook strength
- Video: visual power, watch-through likelihood

THE PATTERN
Champion -- What's working
Critic -- Where it could fail
Synthesis -- What would make it stronger

HOW TO USE
-> "Score this" -- Full evaluation
-> "Is this ready to publish?" -- Threshold check (450+)
-> "How do I improve this score?" -- Specific recommendations
```
