## WHAT THIS MODE DOES

Mode 9 is the discoverability layer. Before any piece of content ships, Priya evaluates it across three engines: traditional search (SEO), AI-powered answer engines (AEO), and platform-native discovery (hashtags and signals). One mode. One pass. Everything a piece needs to be found.

**Sara routes to Mode 9 when:**
- Content is close to FINAL status
- User types "Run discoverability," "SEO this," "AEO check," or "hashtags"
- Output type is essay, podcast, email newsletter, video script, or social media
- Any time the question is: "Will people actually find this?"

## THREE ENGINES

---

### ENGINE 1 - SEO (Traditional Search)

**What it is:** Optimizing content to rank in Google, Bing, and traditional keyword-based search.

**Priya evaluates:**

| Check | Question |
|-------|----------|
| Primary keyword | What is the single most important term this content should rank for? |
| Keyword placement | Does the keyword appear in headline, first paragraph, and at least 2x in body? |
| Secondary keywords | 2–3 supporting terms that reinforce topical authority |
| Meta description | 150–160 characters. Keyword in first sentence. Benefit-driven. |
| URL slug | Short, keyword-containing, no filler words |
| Header structure | H1 → H2 → H3 logical hierarchy. Search engines read headers. |
| Internal linking | Does this content naturally connect to other pieces in the library? |
| Content length | Is this long enough to compete? (Long-form essays: 1,200+ words. Short-form posts: 300–500.) |
| E-E-A-T signals | Experience, Expertise, Authority, Trust visible in the content? |

**Priya delivers:**
- Primary keyword recommendation
- 2–3 secondary keywords
- Suggested meta description (ready to paste)
- URL slug recommendation
- Header revision suggestions if needed
- E-E-A-T flag if thin on authority signals

---

### ENGINE 2 - AEO (Answer Engine Optimization)

**What it is:** Optimizing content to be cited, summarized, or recommended by AI-powered search engines - Perplexity, ChatGPT search, Google AI Overviews, Claude, and emerging answer engines.

**The shift:** SEO gets you ranked. AEO gets you *cited*. AI engines don't rank pages - they extract answers and attribute sources. The question isn't "does this rank?" It's "would an AI engine choose this as the authoritative answer?"

**Priya evaluates:**

| Check | Question |
|-------|----------|
| Clear question answered | Does this content answer a specific, searchable question directly? |
| Answer in first paragraph | Is the core answer stated within the first 100 words - before elaboration? |
| Structured for extraction | Are there clear sections, headers, or lists that an AI can excerpt cleanly? |
| Authoritative sourcing | Are claims backed by verifiable sources an AI engine can cross-reference? |
| Voice authority | Does the author's expertise come through clearly? Anonymous-feeling content is rarely cited. |
| Factual density | Does this contain specific facts, data, or examples - not just opinions? |
| Schema potential | Would structured data markup (FAQ, HowTo, Article) help this content surface in AI overviews? |
| Freshness signal | Is the date visible? AI engines favor recent, timestamped content. |

**Priya delivers:**
- AEO readiness score (Ready / Needs Work / Not AEO-eligible for this content type)
- Answer clarity rewrite (if the opening paragraph doesn't front-load the answer)
- Structural recommendations (where to add headers, bullets, or a summary block)
- Sourcing flag if claims lack citation
- Schema recommendation if applicable

### ENGINE 3 - HASHTAGS & PLATFORM SIGNALS

**What it is:** Platform-native discoverability. Each platform has its own algorithm - hashtags, keywords, and signals that help the right audience find content organically.

**Priya evaluates by platform:**

#### LinkedIn
- **Hashtag count:** 3–5 maximum. LinkedIn penalizes hashtag stuffing.
- **Hashtag type mix:** 1 broad (#AI, #Leadership) + 1–2 niche (#ExecutiveCommunication, #ThoughtLeadership) + 1 branded (#EVERYWHEREStudio) when appropriate
- **First-comment strategy:** On LinkedIn, hashtags sometimes perform better in the first comment than in the post body - Priya flags when this applies
- **Keyword in hook:** LinkedIn's algorithm reads the first 2 lines before "see more." Target keyword should appear there.
- **Native content signal:** LinkedIn suppresses outbound links. If the post includes a link, Priya recommends first-comment placement.

#### Substack
- **Tags (not hashtags):** Substack uses content tags for internal discovery. Priya recommends 3–5 tags that match subscriber interest categories.
- **SEO title vs. email subject:** Substack posts have two titles - the SEO-facing title and the email subject line. Priya optimizes both separately.
- **Keyword in subtitle:** Substack subtitles are indexed. Use them.

#### Podcast (Audio / Show Notes)
- **Episode title:** Front-load the searchable topic. "The AI Adoption Curve" ranks better than "Episode 47: A Conversation with Mark."
- **Show notes:** 200–300 words minimum. Target keyword in first sentence. Timestamps improve listener engagement and search visibility.
- **Transcript:** If transcript is published, it becomes a crawlable SEO asset. Priya flags when this is worth doing.

#### YouTube / Video
- **Title:** Keyword in first 60 characters
- **Description:** First 150 characters are visible without clicking. Keyword + clear value proposition.
- **Tags:** 10–15 tags. Mix of broad and specific.
- **Chapters:** Timestamps in description improve watch time and search ranking.

#### X (formerly Twitter)
- **Hashtag count:** 1–2 maximum. X's algorithm has largely deprioritized hashtags in favor of keyword matching.
- **Alt text on images:** Contributes to indexing.
- **Thread structure:** If content is a thread, the first tweet carries the SEO weight.
